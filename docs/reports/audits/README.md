## File Naming Convention

### Sensitive Reports (Gitignored)

These files contain credentials/PII and are **automatically excluded from git**:

- `EXECUTIVE_SUMMARY.md` - Contains credential snippets
- `RISK_PRIORITY_MATRIX.md` - Contains credential values
- `TECHNICAL_DEEP_DIVE.md` - Contains detailed security info
- `CREDENTIAL_EXPOSURE_REPORT.md` - Contains all credentials
- `index.html` - Interactive dashboard with credential snippets
- `DEBUGGABILITY_OBSERVABILITY_ASSESSMENT.md` - Contains configuration details
- `OBSERVABILITY_EXECUTIVE_SUMMARY.md` - Contains metrics with potential PII
- `SRE_*.md` - Contains infrastructure details

**Convention:** Use `*.priv.md` or `*.private.md` for any new sensitive reports

### Safe Reports (Can be committed)

To create a shareable version of a report:

1. **Redact sensitive data:**
   ```bash
   # Create redacted version
   cat EXECUTIVE_SUMMARY.md | \
     sed 's/rpa_[A-Za-z0-9_]*/rpa_REDACTED/g' | \
     sed 's/glsa_[A-Za-z0-9_]*/glsa_REDACTED/g' | \
     sed 's/[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}/REDACTED_IP/g' \
     > EXECUTIVE_SUMMARY_REDACTED.md
   ```

2. **Use the `_REDACTED.md` suffix** - These are allowed in git (see .gitignore)

## Best Practices

### For Sensitive Reports

1. âœ… **Keep local only** - Don't commit to git
2. âœ… **Encrypt if storing** - Use GPG or disk encryption
3. âœ… **Share securely** - Use encrypted channels (not email)
4. âœ… **Limit access** - Need-to-know basis only
5. âœ… **Create redacted versions** for public/team sharing

### For Sharing Audit Results

**Option 1: Redacted Version**
```bash
# Create shareable version (see redaction script above)
cat REPORT.md | sed 's/credentials/REDACTED/g' > REPORT_REDACTED.md
git add docs/reports/audits/*_REDACTED.md
```

**Option 2: Summary Only**
- Share high-level scores and recommendations
- Omit specific credential values and infrastructure details
- Create custom summary: `AUDIT_SUMMARY_PUBLIC.md`

## .gitignore Protection

All sensitive reports are protected by patterns in `.gitignore`:

```gitignore
# Specific sensitive files
docs/reports/audits/EXECUTIVE_SUMMARY.md
docs/reports/audits/RISK_PRIORITY_MATRIX.md
# ... etc

# General patterns for sensitive docs
*.priv
*.priv.md
*.private.md
*.confidential.md

# Allow redacted versions
!docs/reports/audits/*_REDACTED.md
```
